SOVEREIGN AI — THE BRIEF

BOARD OVERVIEW
FOR THE NEWCOMER — what you're actually looking at

Sovereign AI means owning the whole intelligence pipeline — the models you route to, the data they touch, and the hardware they run on — instead of renting one vendor's black box. ZeusApollo is that idea, built and running: a 19-node homelab doing the work of a cloud platform team for about $0.35/day.

🧭 Own the Route

  • Every request goes to the cheapest model that can do the job — five models, one gateway, no bridge tax.
  • No single-vendor lock-in; swap any model without a rebuild.

🔒 Private by Default

  • Inference and data stay on your own Proxmox fabric — not a third party's logs.
  • Sovereignty is a security posture, and a compliance one.

🤖 Provably Autonomous

  • 31 cron jobs run briefings, backups, and watchdogs 24/7 with no human in the loop.
  • The proof is live further down — not a slide.

🛡️ Resilient by Design

  • Every model has a fallback, every backup an encrypted offsite, every DNS a VRRP twin.
  • 7.2× cheaper than an Opus-primary baseline — and hardened.

WHY IT MATTERS: the levers a board cares about — cost, vendor risk, data privacy, and operational continuity — are exactly what a sovereign stack puts back under your control.

PROOF OF LIFE — HOW IT'S MEASURED

METHODOLOGY
NO VANITY METRICS — the math behind the readouts

Every number above is derived from the fleet itself. A cron writes a status.json snapshot from the live sources below; the page reads it and advances the figures between syncs so the panel never looks frozen.

⏱️ Fleet Uptime

  • Seconds since the last full-fleet restart, ticking live in your browser.
  • Source: Proxmox guest uptime.

🔢 Tokens Routed

  • Cumulative tokens dispatched across all five providers, lifetime.
  • Source: provider usage logs, summed by the balance-tracker cron.

💰 Saved vs Opus

  • What the same work would cost on an Opus-primary baseline, minus actual spend.
  • Formula: Σ(tokens × Opus rate) − Σ(tokens × routed rate).

🖥️ Fleet Nodes

  • Live count of Proxmox guests across Zeus + Apollo.
  • Source: Proxmox VE API inventory.

⚙️ Crons Active

  • Scheduled autonomous jobs currently armed.
  • Source: crontab + systemd timers across the fleet.

🔥 Daily Burn

  • Trailing 24-hour API spend, rolled up each night (now ~$0.35, was ~$4).
  • Source: provider billing, summed by cron.

LIVE FROM THE FLEET: these readouts are written to status.json by a cron on the cluster and read straight into this panel — uptime ticks from the last restart; the rest refresh at each sync. Nothing here is fabricated. Addresses are redacted for OPSEC. And this whole page is open source — the proof includes its own code.

LAW 01

Fly It Before You Trust It

Nothing earns a place in the fleet on theory. Deploy the prototype against a real workload first — production is the only honest flight test.

One Capability, Done Cold

Every service does exactly one job and does it reliably and repeatably. No grab-bag boxes. If it can't be named in a sentence, it isn't built yet.
